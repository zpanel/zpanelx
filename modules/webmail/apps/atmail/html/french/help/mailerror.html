Une erreur de connexion a été reçue lors de l'accès au serveur de messagerie - Le nom de domaine existe-t-il et le serveur correspondant accepte-t-il les connexions entrantes ?

Ceci est causé par des paramètres du serveur de messagerie incorrects. Pour régler ce problème, faites ce qui suit:

  1. Cliquez sur le lien "Login Externe" sur la page de login
  2. Sur la page de login qui va apparaître, entrez les détails de l'adresse complète du serveur de messagerie qui abrite votre compte email. Par exemple mail.yourdomain.com, mail.ispserver.com, etc... Pour trouver l'adresse correcte du serveur de messagerie, vérifiez vos paramètres de messagerie client ou bien contactez votre fournisseur d'accès Internet.
  3. Entrez votre nom d'utilisateur et mot de passe
  4. Cliquez sur "Login"

J'obtiens l'erreur suivante lorsque j'utilise Outlook - le relayage n'est pas autorisé (relaying denied).

Pour résoudre cela, faites ce qui suit:

  1. Ouvrir Outlook
  2. Cliquez sur Outils > Comptes
  3. Cliquez sur l'onglet "Email"
  4. Choisissez votre compte et ensuite choisissez l'onglet "Serveur"

Ici, l'information serveur devrait être comme suit:

Entrant: pop.mondomaine.com
SMTP: smtp.mondomaine.com

Nom du compte : vous devez taper votre adresse email complète, en incluant le nom de domaine

Lorsque j'effectue un login, je reçois une erreur spécifique - "Impossible d'ouvrir le fichier: 'unetable.MYD'"

Il est possible que la base de données sur votre serveur soit corrompue. Si cela se produit, contactez l'Administrateur du système et demandez-lui de réparer la base de données.
